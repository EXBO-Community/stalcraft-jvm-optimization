# Параметры конфигурации

Перед ручной настройкой JVM важно понимать, что этот раздел носит рекомендательный характер.
Приведённые значения в таблицах являются усреднёнными и не учитывают особенности конкретной системы.
Каждое изменение параметра должно быть осознанным - необходимо понимать его назначение и возможные последствия.
В противном случае ручную настройку лучше не выполнять и использовать готовые профили или default.json,
автоматически сформированный под вашу систему утилитой.

Дополнительно ознакомьтесь с ресурсами Oracle:

- [G1 GC tuning](https://docs.oracle.com/javase/8/docs/technotes/guides/vm/gctuning/g1_gc_tuning.html)
- [Список JVM/HotSpot опций](https://docs.oracle.com/javase/8/docs/technotes/tools/windows/java.html)
- [Общий гайд по JVM options](https://www.oracle.com/java/technologies/javase/vmoptions-jsp.html)

Изучение исходного кода OpenJDK дас полное понимания происходящего:

- [OpenJDK source](https://github.com/openjdk/jdk)
- [JEP docs](https://openjdk.org/jeps/)

## Память

| Параметр        | Описание                                                              | Тип       | Рекомендация                                                                                                                                     |
| --------------- | --------------------------------------------------------------------- | --------- | ------------------------------------------------------------------------------------------------------------------------------------------------ |
| `heap_size_gb`  | задаёт фиксированный размер heap-памяти JVM (Xmx/Xms) в гигабайтах.   | `int`     | ~8-12 ГБ при 16+ ГБ свободной памяти ОЗУ. Не забывайте про фоновое потребление ОЗУ другими процессами.                                           |
| `pre_touch`     | Заставляет JVM заранее выделить всю heap-память при старте процесса.  | `boolean` | Выставлять `true`. Снижает случайные микрофризы при выделении памяти, но увеличивает время запуска игры.                                         |
| `metaspace_mb`  | Задаёт начальный размер Metaspace.                                    | `int`     | ~256–512 MB (иногда до 768 MB), чтобы избежать частых расширений Metaspace и связанных с ними микрофризов. Повышать совместно с `heap_size_gb`.  |

## G1GC — основные

| Параметр                              | Описание                                                                                                                        | Тип   | Рекомендация                                                                                                                                   |
| ------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- | ----- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| `max_gc_pause_millis`                 | Задаёт целевую максимальную паузу сборщика мусора в миллисекундах                                                               | `int` | ~20-50. Меньшее значение делает GC агрессивнее и выравнивает FPS, большее значение уменьшает нагрузку на ЦП, но повышает риск заметных фризов. |
| `g1_heap_region_size_mb`              | Задаёт размер одного региона heap в G1 GC, из которых строится управление памятью.                                              | `int` | Не изменять, оставлять авто. При ручной настройке смотрите на свой размер heap. Можно поставить ~4-16.                                         |
| `g1_new_size_percent`                 | Задаёт минимальную долю heap, выделяемую под young generation в G1 GC.                                                          | `int` | ~20%, снижает риск длинных пауз GC.                                                                                                            |
| `g1_max_new_size_percent`             | Задаёт максимальную долю heap, которую может занимать young generation в G1 GC.                                                 | `int` | ~40%, даёт GC гибкость работы под нагрузкой.                                                                                                   |
| `g1_reserve_percent`                  | Задаёт долю heap, резервируемую G1 GC для предотвращения нехватки памяти при пиковых нагрузках.                                 | `int` | ~15-20%, даёт хороший запас на случай пиковых аллокаций.                                                                                       |
| `g1_heap_waste_percent`               | Задаёт допустимый процент  неиспользуемого heap-пространства, после которого G1 GC начинает активнее проводить уборку регионов. | `int` | ~5%-10%, так достигается баланс между плавностью и нагрузкой.                                                                                  |
| `g1_mixed_gc_count_target`            | задаёт целевое число mixed GC-циклов в G1 GC.                                                                                   | `int` | ~2-4, выравнивается фреймтайм.                                                                                                                 |
| `initiating_heap_occupancy_percent`   | Задаёт порог заполнения heap, при котором G1 GC начинает concurrent marking.                                                    | `int` | ~25%-35%. Значения ниже могут дать рваный фреймтайм, а значения выше - дать фризы.                                                             |
| `g1_mixed_gc_live_threshold_percent`  | Включать регион в mixed GC только если в нём < X% живых объектов                                                                | `int` | ~65%-85%, чтобы не попасть на большие и продолжительные фризы.                                                                                 |
| `g1_rset_updating_pause_time_percent` | Задаёт долю времени GC-паузы, которую G1 может тратить на обновление `rs_hot`.                                                  | `int` | ~5%-10%. Меньшее значение делает паузы ровнее.<br>Большее значение снижает overhead GC, но может приводить к более заметным пиковым стопам.    |
| `survivor_ratio`                      | Задаёт соотношение между Eden и Survivor-подобластями в young generation.                                                       | `int` | 6-8. Уменьшает давление на minor GC и снижает риск микрофризов.                                                                                |
| `max_tenuring_threshold`              | Задаёт максимальное количество сборок молодого поколения, после которых объект переносится в old gen.                           | `int` | ~4-8. Баланс между частыми перемещениями и удержанием объектов в young gen.<br>Можно попробовать 1.                                            |

## G1GC — продвинутые (минимизация STW)

| Параметр                                     | Описание                                                                                                                   | Тип       | Рекомендация                                                                                                                                                                           |
| -------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------- | --------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `g1_satb_buffer_enqueuing_threshold_percent` | Задаёт порог заполнения SATB-буфера, при котором G1 GC начинает активнее его обрабатывать.                                 | `int`     | ~30-60, при необходимости слегка повысьте для уменьшения GC-нагрузки в фоне.<br>0 - Отключено.                                                                                         |
| `g1_conc_rs_hot_card_limit`                  | Задаёт порог "горячих" карточек памяти, при котором G1 GC активнее обрабатывает изменения ссылок между регионами.          | `int`     | ~16-32 можно слегка увеличить для снижения фоновой GC-активности.<br>0 - Отключено.                                                                                                    |
| `g1_conc_refinement_service_interval_millis` | Задаёт интервал в миллисекундах между циклами фоновой обработки `rs_hot`.                                                  | `int`     | ~10–20, снижение значения делает GC более отзывчивым, но увеличивает фоновую нагрузку на CPU.<br>0 - Отключено                                                                         |
| `gc_time_ratio`                              | Задаёт целевое соотношение времени, которое JVM допускает тратить на сборку мусора относительно времени работы приложения. | `int`     | Оставлять дефолт.<br>0 - Отключено.                                                                                                                                                    |
| `use_dynamic_number_of_gc_threads`           | Позволяет JVM автоматически изменять количество потоков сборщика мусора в зависимости от нагрузки.                         | `boolean` | Выставлять `true`. Обычно улучшает стабильность фреймтаймов, так как GC подстраивается под нагрузку, но на слабых CPU может иногда добавлять небольшую вариативность задержек.         |
| `use_string_deduplication`                   | Включает дедупликацию строк в G1 GC, объединяя одинаковые строки в одну копию в памяти.                                    | `boolean` | Выставлять `true`, снижает потребление heap-памяти и может уменьшить давление на GC, но добавляет небольшой фоновый CPU-оверход, обычно это выгодный размен для долгих игровых сессий. |

## Потоки GC

| Параметр                        | Описание                                                                                                              | Тип   | Рекомендация                                                                                                                                                  |
| ------------------------------- | --------------------------------------------------------------------------------------------------------------------- | ----- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `parallel_gc_threads`           | Задаёт количество потоков, используемых сборщиком мусора для параллельной работы.                                     | `int` | -1 - автоматическое определение.<br>Как правило выставлять значение, равное значению физических ядер ЦП.                                                      |
| `conc_gc_threads`               | Задаёт количество потоков для фоновых фаз сборщика мусора.                                                            | `int` | Выставлять ~25% от `parallel_gc_threads`, округляя в меньшую сторону. Но не меньше 1.                                                                         |
| `soft_ref_lru_policy_ms_per_mb` | Задаёт, как долго JVM "терпит" soft references в памяти перед их очисткой (в миллисекундах на 1 MB свободной памяти). | `int` | ~50–200 для снижения частых очисток кэшей и уменьшения микрофризов. Но слишком большие значения могут раздувать использование памяти и повышать риск GC-пауз. |

## JIT-компиляция

| Параметр                      | Описание                                                                                                            | Тип       | Рекомендация                                                                                                                                                                             |
| ----------------------------- | ------------------------------------------------------------------------------------------------------------------- | --------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `reserved_code_cache_size_mb` | Задаёт максимальный объём памяти, выделяемой JVM под кэш скомпилированного JIT-кода в МБ.                           | `int`     | Для долгоживущих приложений как STALCRAFT: X рекомендуется выставлять ~256-512 MB.                                                                                                       |
| `max_inline_level`            | Ограничивает глубину вложенного inlining’а методов при JIT-компиляции.                                              | `int`     | ~12-15 для CPU-нагруженных приложений. Чем выше - тем больше надо кеша JIT.                                                                                                              |
| `freq_inline_size`            | Максимальный размер метода, который JVM считает "часто вызываемым" и может агрессивно инлайнить.                    | `int`     | ~300-500 для горячих участков кода. Чем выше - тем больше надо кеша JIT.                                                                                                                 |
| `inline_small_code`           | Задаёт порог размера, ниже которого скомпилированный метод считается "маленьким" и легче инлайнится.                | `int`     | ~2000–3000 для повышения агрессивности inlining. Значение 0 выключает порог. Чем выше - тем больше надо кеша JIT.                                                                        |
| `max_node_limit`              | Ограничивает максимальное количество узлов в IR-графе компилятора C2 для одного метода.                             | `int`     | ~100 000-240 000. Значения выше могут дать микрофризы.                                                                                                                                   |
| `node_limit_fudge_factor`     | Задаёт допуск сверх `max_node_limit`, позволяя компилятору повышать его при оптимизациях.                           | `int`     | ~2000-8000. Изменять только совместно с `max_node_limit`. <br>При `max_node_limit`= 240000 можно выставить 8000.<br>0 - Отключает повышение. Высокие значения могут вызывать микрофризы. |
| `nmethod_sweep_activity`      | Регулирует интенсивность очистки code cache от устаревших JIT-скомпилированных методов.                             | `int`     | ~1-4 для стабильности при долгой игре. Cлишком агрессивная очистка может вызывать повторные компиляции и просадки FPS.                                                                   |
| `dont_compile_huge_methods`   | Pапрещает JIT-компиляцию чрезмерно больших методов.                                                                 | `boolean` | Выставлять `false` на очень мощных ЦП. Во всех остальных случаях `true`.                                                                                                                 |
| `allocate_prefetch_style`     | Задаёт стратегию аппаратного/программного prefetch при выделении памяти объектам в JVM.                             | `int`     | ~1-3 в зависимости от мощности ЦП. Снижает задержки доступа к памяти и может слегка улучшить плавность кадров.<br>0 - стандартная стратегия.                                             |
| `always_act_as_server_class`  | Заставляет JVM вести себя как серверная машина и использовать серверные JIT-оптимизации даже на клиентской системе. | `boolean` | Выставлять `true` на многоядерных (8+) ЦП. В остальных случаях `false`. Возможно увеличение времени запуска игры.                                                                        |
| `use_xmm_for_array_copy`      | Включает использование SIMD для ускоренного копирования массивов в JVM.                                             | `boolean` | Выставлять `true`, улучшает фреймтаймы, снижает нагрузку на ЦП. Кроме древних процессоров.                                                                                               |
| `use_fpu_for_spilling`        | Позволяет JVM использовать FPU/SSE регистры для временного хранения значений при нехватке регистров CPU.            | `boolean` | Выставлять `true`. Разгружает основные регистры, стабилизирует фреймтайм в ЦП-нагруженных сценах.                                                                                        |
