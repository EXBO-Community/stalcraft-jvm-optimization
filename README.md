# Stalcraft JVM Wrapper

[![eng](https://img.shields.io/badge/lang-English-blue)](README.en.md)
[![ru](https://img.shields.io/badge/lang-Russian-blue)](README.md)

**Утилита для модификации параметров запуска JVM и оптимизации её работы.**

**JVM (Java Virtual Machine)** — это среда выполнения, через которую работает [STALCRAFT: X](https://stalcraft.ru/).

Код игры исполняется не напрямую на системе, а внутри виртуальной машины Java. Во время работы она компилирует его
в машинный код под конкретный ПК (JIT-компиляция). Фактически это дополнительный слой между игрой и железом,
который отвечает за выполнение кода и его адаптацию под систему.

Данная программа позволяет изменить параметры запуска JVM для повышения производительности игры, используя как предустановленные,
так и пользовательские JSON-файлы конфигурации.

> [!WARNING]
> Данный проект является **неофициальной** утилитой, разработанной [SilentBless](https://github.com/SilentBless).
> Утилита не аффилирована с EXBO, однако была проверена [GloomyFolken](https://github.com/GloomyFolken)
> и классифицирована как безопасное ПО.

> [!IMPORTANT]
> Утилита не модифицирует параметры запуска JVM для систем с объёмом оперативной памяти 8 ГБ и меньше.
> Агрессивная оптимизация при малом объёме памяти может навредить вашему ПК.
> Вместо этого используйте стандартные настройки лаунчера EXBO.

## Вносимые изменения

Утилита (wrapper) выполняет перехват запуска процесса игры `stalcraft.exe` (лаунчер) или `stalcraftw.exe` (Steam) для:

- **Подбора оптимальной конфигурации JVM:** объём выделенных ресурсов, режим работы GC (Garbage Collector) и JIT-компиляции.
- **Повышения приоритета процесса игры:** процесс выполняется приоритетнее по сравнению с другими процессами.

Wrapper устанавливается **один раз** и автоматически запускается при каждом запуске игры.

> [!IMPORTANT]
> Файлы игры при этом не затрагиваются и не модифицируются.
> Утилита также не вмешивается в работу процесса игры и не встраивается в него.

## Требования

- **Операционная система:** Windows 10/11
- **Версия игры:** Steam/Лаунчер/EGS
- **Права ОС:** права администратора в Windows
- **ЦП:** 4 и более ядер
- **ОЗУ:** 12+ ГБ

## Работа с утилитой

### Установка

1. Добавьте папку с игрой в исключения Защитника Windows или другого Антивирусного ПО:
   - Пример для Steam: `C:\Program Files\Steam\steamapps\common\STALCRAFT`
   - Пример для Лаунчера: `C:\Users\User\AppData\Roaming\EXBO`
2. Создайте в папке с игрой каталог `jvm_wrapper`.
3. Скачайте [последнюю версию](../../releases/latest) `wrapper.exe` в папку `jvm_wrapper`.
4. Запустите утилиту от имени администратора.
5. В открывшемся меню выберите пункт `Install` при помощи стрелок и нажмите **Enter**.

**Теперь можно запускать игру!**

> [!NOTE]
> Утилита не распространяется на другие приложения, использующие JVM.

### Удаление

1. Запустите утилиту от имени администратора.
2. В открывшемся меню выберите пункт `Uninstall` при помощи стрелок и нажмите **Enter**.
3. Перейдите в папку с игрой.
4. Удалите папку `jvm_wrapper`.
5. Перезапустите игру, если она запущена.

### Конфигурация

После установки утилита автоматически создаст профиль конфигурации `default.json`,
который будет располагаться в папке `jvm_wrapper/configs/default.json`.
Именно с этим профилем игра будет запускаться по умолчанию.
Данный профиль будет адаптирован под параметры вашего компьютера, но его наличие не исключает возможность кастомной настройки.

**Конфигурация сохраняется в реестре Windows:** `HKCU\\Software\\StalcraftWrapper`.

Конфигурацию запуска можно менять самостоятельно. Для этого:

1. Запустите утилиту от имени администратора.
2. В открывшемся меню выберите пункт `Select Config` при помощи стрелок и нажмите **Enter**.
3. Выберите необходимый файл конфигурации и нажмите **Enter**.
4. Перезапустите игру, если она запущена.

> [!NOTE]
> По умолчанию доступна только конфигурация `default.json`, однако она *не является* единственой опцией.
> В разделах [Пресеты конфигурации](#Пресеты-конфигурации) и [Кастомная конфигурация](#Кастомная-конфигурация)
> представлены необходимые инструкции.

#### Пресеты конфигурации

В репозитории утилиты представлены готовые пресеты конфигурации под ПК разных характеристик.
Ниже приведена описательная таблица для каждого из пресетов:

| Профиль       | Требования                   | Описание                                                                                                                                                        |
| ------------- | ---------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `weak.json`   | ЦП: 4+ ядра<br>ОЗУ: ~8-12 ГБ | Профиль для ПК "слабого сегмента". Если ваш ПК не является "картошкой", но всё таки достаточно слабенький, то это может подойти для вас.                        |
| `medium.json` | ЦП: 6+ ядер<br>ОЗУ: ~16 ГБ   | Сбалансированный профиль для ПК "среднего сегмента". Подходит для большинства современных систем.                                                               |
| `max.json`    | ЦП: 8+ ядер<br>ОЗУ: ~32+ ГБ  | Профиль для ПК "производительного сегмента". Подходит для получения максимальной производительности JVM, сохраняя стабильность её работы в рамках STALCRAFT: X. |

Чтобы использовать какой-либо из пресетов, перейдите в каталог репозитория [`/configs`](./configs/),
откуда можно скачать интересующий вас профиль в папку `jvm_wrapper/configs`.

Запустите утилиту, выберете `Select Config` в меню. Теперь в списке, помимо `default.json`, должен появиться
еще один профиль конфигурации. Выберете его, после чего перезапустите игру.

#### Кастомная конфигурация

Для создания своего собственного профиля конфигурации достаточно скопировать файл `default.json`,
переименовать его, например, в `my_setup.json`, после чего отредактировать через любой доступный
тестовый редактор.

> [!CAUTION]
> Кастомная конфигурация рекомендуется только тем, кто **на 100% понимает**, что он делает.
> В противном случае вы рискуете нарушить не только стабильность JVM и, как следствие, игры, но
> и всей операционной системы.

Создание своей собственной конфигурации должно сопровождаться изучением [документации](./docs/PARAMS.md)
к параметрам конфигурации.

> [!TIP]
> Если вы описали кастомную конфигурацию в `default.json` и хотите вернуться
> к рекомендованным настройкам — выберите в меню пункт `Regenerate Config`.
> Данное действие запишет оптимальные настройки для вашего ПК в `default.json`.

## Дополнительно

### Large Pages

**Large Pages** — это режим работы виртуальной памяти, при котором используются страницы большего размера, чем стандартные 4 KB.

Включение Large Pages уменьшает накладные расходы на работу с памятью, как следтствие делают GC и доступ к heap стабильнее и быстрее.
Это происходит из-за непрямой работы ЦП с ОЗУ вашего компьютера. ЦП использует TLB (Translation Lookaside Buffer).

> [!CAUTION]
> Large Pages закрепляют память за приложением и не позволяют системе перераспределять её.
> Неправильная настройка может привести к нестабильной работе ОС. Отдавайте отчёт своим действиям!
> Убедитесь, что выделенная память в профиле конфигурации не привышает 40%-50% от общего обьема ОЗУ,
> а общий запас ОЗУ 16+ ГБ.

Для того, чтобы включить Large Pages, следуюте следующим шагам:

1. Нажмите сочетание клавиш `win` + `R`.
2. Введите `secpol.msc` и нажмите `Enter`.
3. Перейдите *Локальные политики → Назначение прав пользователя*.
4. Найдите политику *"Блокировка страниц в памяти"*.
5. Откройте политику двойным нажатием, добавьте своего пользователя или группу "Администраторы".
6. Примените изменения и перезагрузите ПК.
